# GreenSync Report Generation Skill

## 標準構成

1. 目的
2. 結論
3. 調査方法
4. 調査結果
   - 4-1. 個別項目
   - 4-2. 個別項目
5. 結果整理・考察
6. 提言または最終結論

## 必須ルール

- Markdownを正本とする。
- 文書ID、タイトル、版数、調査日、ステータスを冒頭に記載する。
- 結論には、調査全体から導かれる主要判断を記載する。
- 調査方法には、対象範囲、情報源、比較軸、制約を記載する。
- 個別企業・プロジェクトには公式リンクを付ける。
- 事実、推測、評価、提言を区別する。
- GreenSyncとの重複領域、差別化候補、連携可能性を記載する。
- 略語は初出時に正式名称と意味を記載する。
- 既存報告書更新時は版数と変更履歴を更新する。
