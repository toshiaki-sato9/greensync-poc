# Wi-Fi・MQTT再接続 動作確認手順

## 1. 目的

Wi-FiまたはMQTT Brokerが利用できない場合も、GreenSyncのメインループが停止せず、水分計測、散水終了、BtnA緊急停止を継続することを確認する。通信復旧後は自動再接続し、MQTT購読とHome Assistant Discoveryを復元することを確認する。

## 2. PASS条件

- Wi-Fi未接続でもデバイスが起動処理からメインループへ進む。
- Wi-Fi再接続は10秒間隔でバックグラウンド実行される。
- MQTT再接続は5秒間隔で1回ずつ実行される。
- 通信障害中も10秒周期のセンサログが継続する。
- 通信障害中も散水開始から3秒後にポンプが停止する。
- 通信障害中もBtnA長押しでポンプが停止する。
- Wi-Fi・Broker復旧後に自動再接続する。
- MQTT再接続後に閾値トピックを再購読する。
- MQTT再接続後にDiscoveryと閾値状態を再送する。
- 接続試行ループに固定 `delay()` がなく、成功まで処理を占有しない。

## 3. 事前準備

```bash
cd firmware/atom-s3-lite
pio run
```

実機へ書き込み、シリアルログを監視する。

```bash
bash build.sh
```

Broker側では次をListenする。

```bash
mosquitto_sub -h BROKER_HOST -p 1883 -v \
  -t 'homeassistant/#' \
  -t 'greensync/#'
```

## 4. テストケース

### NET-001: 通常起動

1. Wi-FiとBrokerを起動した状態でデバイスを再起動する。
2. シリアルログを確認する。

期待結果:

- `WiFi connection attempt` の後に `WiFi connected` が表示される。
- `MQTT connection attempt` の後に `MQTT connected` が表示される。
- subscribe結果が `OK` になる。
- Discovery summaryが `ALL OK` になる。

### NET-002: Wi-Fi停止状態で起動

1. 対象SSIDのアクセスポイントを停止する。
2. デバイスを再起動する。
3. 30秒以上ログを取得する。

期待結果:

- setupがWi-Fi接続待ちで停止しない。
- 約10秒間隔で接続試行ログが出る。
- センサログが約10秒周期で継続する。
- MQTT接続はWi-Fi復旧まで開始しない。

### NET-003: Wi-Fi切断・復旧

1. 正常接続後にアクセスポイントを停止する。
2. 30秒待機する。
3. アクセスポイントを復旧する。

期待結果:

- 切断を1回ログ出力する。
- センサ処理が継続する。
- 復旧後にIPアドレスとRSSIをログ出力する。
- MQTTも自動再接続する。
- Discoveryと閾値状態を再送する。

### NET-004: Broker停止状態で起動

1. Wi-Fiは利用可能、Brokerは停止した状態にする。
2. デバイスを再起動する。
3. 20秒以上ログを取得する。

期待結果:

- Wi-Fi接続後、MQTT接続を約5秒間隔で再試行する。
- 1回の失敗後にメインループへ戻る。
- センサログが約10秒周期で継続する。
- MQTT接続成功まで待つ無限ループに入らない。

### NET-005: Broker切断・復旧

1. MQTT接続完了後にBrokerを停止する。
2. 20秒待機する。
3. Brokerを復旧する。

期待結果:

- MQTT切断をログ出力する。
- 5秒間隔で再接続する。
- 復旧後、閾値トピックを再購読する。
- Discovery 4件と閾値状態を再送する。
- Home Assistantのデバイスとエンティティが復旧する。

### SAFE-001: 通信障害中の散水完了

最初は水を接続せず、LEDまたはテスターでポンプGPIOを確認する。

1. 水分率が閾値未満になる条件を作り、散水を開始させる。
2. 散水中にBrokerまたはWi-Fiを停止する。
3. ポンプ出力時間を測定する。

期待結果:

- 通信障害に関係なく、散水開始から約3秒でポンプがOFFになる。
- 制御状態が `IDLE` へ戻る。
- 通信復旧後、ポンプ停止状態をMQTTへ送信する。

### SAFE-002: 通信障害中の緊急停止

1. BrokerまたはWi-Fiを停止する。
2. 散水中にBtnAを1.5秒長押しする。

期待結果:

- ポンプが直ちにOFFになる。
- `EMERGENCY_STOP` へ遷移する。
- 通信障害中も自動散水を再開しない。
- 通信復旧後、ポンプ停止状態をMQTTへ送信する。
